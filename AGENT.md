# qsd Admin 项目规范

## 项目定位

这是运单管理与站点内容运营的一体化后台项目。当前主线除了后台内容管理、运单管理、字典管理外，新增一套会员系统，用于支撑小程序侧会员注册、登录、资料维护和会员运单查询。

本仓库当前仍以 `管理后台 + Java API` 为核心，不直接包含小程序前端页面代码。本次会员系统改造的目标是：

- 后台新增会员管理能力
- 后端新增一组面向小程序调用的会员接口
- 运单体系支持按会员维度关联与查询
- 为后续独立开发微信小程序前端提供稳定 API

## 技术约束

- 后台前端统一使用 `Vue 3 + TypeScript + Vite + Vue Router + Pinia`
- UI 方案固定为 `Element Plus + Tailwind CSS`
- 后端统一使用 `Java 21 + Spring Boot 3 + Spring Security + JWT + MyBatis-Plus + Flyway`
- 前端业务请求统一走 `/api`
- 开发环境通过 Vite 代理转发到 `VITE_API_PROXY_TARGET`
- 默认脚手架残留组件、资源和未使用代码不保留，确认无引用后应及时删除

## 后台前端规范

- 根路由 `/` 默认跳转到 `/dashboard`
- 未登录访问后台路由时，统一跳转到 `/login?redirect=当前地址`
- 已登录访问 `/login` 时，直接回到 `/dashboard`
- 登录成功后必须先完成 `/api/auth/me` 和动态菜单注入，再进入目标页
- 后台布局保持左侧菜单与右侧内容区等高
- 只允许右侧内容区内部滚动，不使用全局滚动条
- 前端开发端口默认使用 `5174`

## 运单与会员协同规范

- 运单管理页面位于 `frontend/src/views/WaybillView.vue`
- 后台新增会员管理页面，默认进入“会员列表”，支持查看会员基础资料、状态、绑定运单数量
- 会员和运单必须建立明确关联关系，至少支持：
  - 一个会员可绑定多条运单
  - 运单可通过主运单号、手机号或会员 ID 归属到会员
  - 小程序侧只能查询当前登录会员自身数据
- 运单状态、线路类型、分段状态等基础枚举继续统一收敛到字典模块
- 数据库存储继续保留稳定英文编码值，例如 `created`、`active`、`disabled`
- 前后端显示层统一通过字典标签映射成中文

## 会员系统规范

- 会员系统分为后台管理接口与小程序会员接口两套入口，不能复用后台管理员登录态
- 后台管理员接口继续使用现有 `/api/**` + 管理员 JWT
- 小程序会员接口使用公开可访问入口，但内部使用独立会员 token 认证
- 首期会员能力至少包含：
  - 会员注册
  - 手机号登录
  - 会员资料查询
  - 会员资料更新
  - 会员运单列表查询
  - 会员运单详情查询
- 当前阶段不直接对接真实微信 `code2Session`
- 为便于联调，首期采用“手机号 + 密码 / 验证码占位方案”的自有会员认证模型，并在文档中明确后续可切换微信登录
- 会员状态至少包含：
  - `active`
  - `disabled`
  - `pending`

## 小程序接入规范

- 小程序不得直接调用后台管理员 JWT 接口
- 小程序会员接口统一放在公开会员命名空间，例如：
  - `POST /api/member/auth/register`
  - `POST /api/member/auth/login`
  - `GET /api/member/profile`
  - `PUT /api/member/profile`
  - `GET /api/member/waybills`
  - `GET /api/member/waybills/{id}`
- 返回结构保持稳定，避免把后台管理 DTO 直接暴露给小程序
- 小程序接口默认只返回当前会员可见字段，不暴露后台备注、内部操作信息和其他会员数据

## 后台会员管理规范

- 后台必须提供会员管理页面，支持：
  - 会员列表
  - 关键字搜索
  - 状态筛选
  - 新增会员
  - 编辑会员
  - 启用/禁用会员
  - 查看会员绑定运单
- 会员管理接口位于：
  - `GET /api/admin/members`
  - `GET /api/admin/members/{id}`
  - `POST /api/admin/members`
  - `PUT /api/admin/members/{id}`
  - `PUT /api/admin/members/{id}/status`
- 会员管理菜单需要纳入管理员菜单体系并授予超级管理员默认权限

## 安全与迁移规范

- Spring Security 仅放行明确声明的公开接口
- 必须区分管理员 token 与会员 token，避免互相串用
- Flyway 已执行迁移必须保持不可变，不能直接修改历史迁移文件
- 菜单、权限、会员表、运单会员关联如影响已执行迁移，必须新增迁移文件
- 本地 Windows 启动后端优先使用 `backend/start-dev.ps1` 或 `backend/start-dev.cmd`
- 本地开发使用 `Java 21`

## 验证要求

- 前端改动后至少保证 `npm run build` 通过
- 后端改动后至少保证 `mvn -DskipTests package` 通过
- 涉及登录、权限、路由、会员注册、会员资料、会员运单查询时，需要做对应功能回归
- 会员相关最少补齐：
  - 注册成功路径验证
  - 登录成功路径验证
  - 禁用会员不可登录验证
  - 会员只能读取自身运单验证
