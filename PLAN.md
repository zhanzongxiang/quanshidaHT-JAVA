# qsd Admin Backend Plan

## 目标

本文件定义后台一期的推荐实施顺序，目标是在可控范围内快速搭出一套可用的运营后台，而不是一次性做成全量业务平台。

所有实现边界仍以 `AGENT.md` 为准。

## 阶段 0：项目基础壳子

目标：

- 建立后台项目骨架
- 完成基础布局和导航结构
- 打通登录与权限拦截
- 锁定前后端技术栈与语言规范

交付内容：

- 前端项目：`Vue 3 + TypeScript + Vite + Vue Router + Pinia + Element Plus`
- 后端项目：`Java 21 + Spring Boot 3`
- 数据库与缓存接入约定：`MySQL 8 + Redis`
- 统一接口风格：`REST API`
- 统一语言规范：文档/注释/提交说明中文，标识符英文
- 登录页
- 后台主布局
- 路由系统
- 基础菜单
- 404/403 页面
- 全局请求封装
- 全局错误提示机制

验收点：

- 技术栈不再处于待定状态
- 未登录无法访问后台页面
- 登录后可进入主框架
- 菜单和路由基础可用

## 阶段 1：账号权限

目标：

- 建立后台最基础的用户、角色、权限体系

交付内容：

- 用户列表
- 角色列表
- 菜单权限配置
- 基础操作权限控制

验收点：

- 不同角色可见菜单不同
- 无权限用户不能进入或执行关键操作

## 阶段 2：工作台

目标：

- 提供后台首页总览和待办入口

交付内容：

- 数据总览卡片
- 待处理事项
- 最近操作记录

验收点：

- 工作台能展示基础汇总数据
- 可作为运营和客服的进入首页

## 阶段 3：内容管理

目标：

- 先支撑官网静态内容维护

优先范围：

- 首页内容
- 联系方式
- 页脚信息
- 基础站点信息
- 线路页内容

交付内容：

- 页面内容列表
- 内容编辑页
- 发布/保存能力

验收点：

- 内容可维护
- 字段结构清晰
- 修改路径稳定

## 阶段 4：新闻管理

目标：

- 让新闻模块进入标准化后台维护

交付内容：

- 新闻列表
- 新建/编辑新闻
- 发布状态
- 发布时间
- 封面图与排序

验收点：

- 新闻可新增、编辑、发布、下架
- 新闻数据结构可持续扩展

## 阶段 5：素材库

目标：

- 统一图片与文件资源入口

交付内容：

- 素材上传
- 素材列表
- 分类管理
- 资源引用信息

验收点：

- 内容和新闻可引用素材
- 素材路径稳定

## 阶段 6：线索管理

交付内容：

- 线索详情

- 线索可查询、分配、跟进
- 跟进记录可审计

## 阶段 7：运单管理

目标：

- 支撑运单与轨迹维护

交付内容：

- 轨迹节点管理
- 异常件标记

- 轨迹时间线可编辑、可展示
- 异常状态逻辑清晰

## 阶段 8：客户管理

目标：

- 建立基础客户档案

交付内容：

- 客户列表
- 客户详情
- 关联线索

验收点：

- 客户与线索、运单关系清晰

## 阶段 9：操作日志与系统设置

目标：

- 补齐审计与基础配置能力

交付内容：

- 登录日志
- 操作日志
- 站点设置
- 上传设置
- 基础字典

验收点：

- 关键写操作可追踪
- 基础配置可后台维护

## 实施原则

- 先做壳子和权限，再做业务模块。
- 先做官网运营能力，再做客服支撑能力。
- 先保证 CRUD 可用，再做高级筛选、导出、批量操作。
- 不要在一期提前实现订单、财务、审批和复杂报表。
- 前端固定使用 Vue 技术栈，后端固定使用 Java 技术栈，不在一期内随意换栈。

## 每阶段通用验证

每完成一个阶段，至少验证：

- 项目可构建
- 对应页面路由正常
- 权限控制有效
- 列表加载正常
- 表单校验有效
- 写操作反馈明确
- 关键写操作产生日志
