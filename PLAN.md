# qsd Admin Backend Plan

## 目标

本文档定义后台一期的推荐实施顺序，目标是在可控范围内快速交付一套可用后台，而不是一次性做成完整业务平台。

## 阶段 0：项目基础壳子

目标：

- 建立前后端项目骨架
- 完成基础布局和导航
- 打通登录与权限拦截
- 锁定技术栈与语言规范

交付内容：

- 前端：Vue 3 + TypeScript + Vite + Vue Router + Pinia + Element Plus
- 后端：Java 21 + Spring Boot 3
- 数据库与缓存接入约定：MySQL 8 + Redis
- 登录页
- 后台主布局
- 路由系统
- 基础菜单
- 403/404 页面
- 全局请求封装
- 全局错误提示机制
- `/` 默认重定向到 `/dashboard`
- 未登录访问后台路由时统一跳转到 `/login?redirect=...`
- 已登录访问 `/login` 时自动回到 `/dashboard`
- `404` 页面不作为公开路由绕过权限拦截

验收点：

- 技术栈不再处于待定状态
- 未登录无法访问后台页面
- 登录后可进入主框架
- 菜单和路由基础可用
- 未登录访问 `/` 和 `/dashboard` 时会进入登录页
- 已登录访问 `/login` 时会回到 `/dashboard`
- 已登录访问未知地址时才显示 `404`

## 阶段 1：账号权限

目标：

- 建立后台基础的用户、角色、权限体系

交付内容：

- 用户列表
- 角色列表
- 菜单权限配置
- 基础操作权限控制

验收点：

- 不同角色看到的菜单不同
- 无权限用户不能进入或执行关键操作

## 实施原则

- 先做壳子和权限，再做业务模块
- 先保证 CRUD 可用，再做高级筛选、导出、批量操作
- 一期不提前实现订单、财务、审批和复杂报表
- 前端固定使用 Vue 技术栈，后端固定使用 Java 技术栈

## 通用验证

每完成一个阶段，至少验证：

- 项目可构建
- 对应页面路由正常
- 权限控制有效
- 列表加载正常
- 表单校验有效
- 写操作反馈明确
- 关键写操作产生日志
