# qsd Admin 开发计划

## 阶段 0：基础设施

- 搭建 Vue 3 + TypeScript + Vite 管理端
- 搭建 Spring Boot 3 + Spring Security + JWT + MyBatis-Plus + Flyway 后端
- 打通登录、鉴权、菜单、基础布局

## 阶段 1：后台基础能力

- 用户、角色、菜单、权限
- 登录态恢复与路由守卫
- 内容管理基础结构

## 阶段 2：业务模块

- 首页内容管理
- 新闻管理
- 运单管理
- 字典管理

## 阶段 3：会员系统与小程序接入

### 后端已完成

- 新增 `member_user`

- 新增会员注册、登录、资料、运单查询接口
- 新增后台会员管理接口

- 打通会员与运单的可见关系

### 前端已完成

- 新增会员管理菜单与页面
- 支持会员列表、搜索、状态筛选
- 支持新增、编辑、启停会员

- 仓库内已新增 `uni-app` 小程序联调骨架

## 阶段 4：微信登录与微信支付扩展

### 4.1 会员身份体系扩展

后端计划：

- 在会员表或独立绑定表中增加 `openid`、`unionid`、绑定时间、绑定来源
- 增加 `code2Session` 登录/绑定流程
- 保留手机号登录，支持“微信登录 + 手机号补充资料”双轨模式
- 为会员端 token 增加微信身份上下文

前端/小程序配合计划：

- 小程序登录改为先取 `code` 再换取会员会话
- 增加首次绑定手机号流程
- 会员资料页增加微信绑定状态展示
- 当前已落地：`uni-app` 骨架已提供手机号登录、微信登录入口、资料页

### 4.2 支付域建模

后端计划：

- 新增 `pay_order`
- 新增 `pay_transaction`
- 新增 `refund_order`
- 新增 `pay_notify_log`
- 新增 `refund_notify_log`
- 新增 `pay_reconcile_record`
- 设计支付状态机：待支付、支付中、已支付、退款中、已退款、关闭、异常
- 将支付单与运单、会员、业务场景关联

后台前端计划：

- 新增支付单管理页面
- 新增退款单管理页面
- 新增回调日志与异常单排查页面
- 新增运单详情中的支付信息区块

### 4.3 小程序支付链路

后端计划：

- 提供小程序支付下单接口
- 调用微信支付 JSAPI/小程序下单能力
- 返回 `requestPayment` 所需参数
- 提供支付结果查询接口
- 提供会员支付记录列表接口

小程序配合计划：

- 在会员运单或订单详情中增加支付入口
- 调用 `wx.requestPayment`
- 支付完成后主动轮询支付结果
- 展示支付成功、失败、取消、处理中状态
- 当前已落地：`uni-app` 运单详情页已提供支付准备入口和 `requestPayment` 调用骨架

### 4.4 回调、退款与对账

后端计划：

- 新增微信支付回调接口
- 新增微信退款回调接口
- 增加验签、解密、幂等、防重放
- 增加主动查单任务
- 增加日对账下载、解析、差异比对任务
- 增加退款申请与退款状态同步能力

后台前端计划：

- 支持支付状态筛选、交易号查询、异常单筛选
- 支持发起退款与查看退款状态
- 支持查看回调原文、处理结果、失败原因
- 支持查看对账结果与差异项

### 4.5 配置与安全

- 支付商户配置改为环境变量或安全配置中心
- 区分测试商户与正式商户
- 日志中屏蔽敏感字段
- 增加支付限流、重试、告警
