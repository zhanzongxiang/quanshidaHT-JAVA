# qsd Admin Backend TODO

## 基础设施

- 明确并锁定后台固定技术栈：
  - 前端 `Vue 3 + TypeScript + Vite + Vue Router + Pinia + Element Plus`
  - 后端 `Java 21 + Spring Boot 3`
  - 数据库 `MySQL 8`
  - 缓存 `Redis`
  - 接口风格 `REST API`
- 明确语言规范：
  - 文档、注释、提交说明使用中文
  - 表名、字段名、接口路径、代码标识符使用英文
- 搭建后台项目脚手架与目录结构。
- 确定唯一的后台 UI 组件库，不要混用。
- 建立登录页、主布局、路由系统和权限拦截。
- 建立统一请求层、错误处理和消息反馈机制。
- 设计 403、404、空状态、加载状态等基础页面状态。

## 权限系统

- 设计用户、角色、权限的基础模型。
- 明确菜单权限和关键操作权限的控制方式。
- 建立用户管理页、角色管理页、菜单权限页。
- 明确超级管理员、运营、客服三类角色的默认权限集。

## 工作台

- 设计后台首页的数据总览模块。
- 明确待办事项和最近操作的展示结构。
- 定义工作台第一版指标，不要一开始堆太多图表。

## 内容管理

- 设计官网页面内容的数据结构。
- 拆分页面级内容与区块级内容的关系。
- 优先支持首页、联系信息、页脚和线路页内容维护。
- 设计内容编辑页，避免做成过于复杂的万能页面编辑器。

## 新闻管理

- 设计新闻文章对象结构。
- 支持标题、摘要、封面图、正文/外链、发布时间、状态、排序。
- 评估是否需要分类和标签。
- 确认是否需要后续扩展新闻详情页模板。

## 素材库

- 设计素材上传和分类模型。
- 支持内容管理与新闻管理复用素材。
- 记录素材 URL、类型、大小、上传时间、引用关系。

## 线索管理

- 设计线索对象与跟进记录对象。
- 明确线索来源、状态、负责人、备注结构。
- 支持线索分配、跟进、状态流转。
- 预留线索转客户的关系设计，但一期不必做复杂转化流程。

## 运单管理

- 设计运单对象与轨迹节点对象。
- 明确运单状态、异常状态、签收状态的枚举。
- 支持轨迹时间线编辑。
- 支持异常件标记与说明。

## 客户管理

- 设计客户基本资料对象。
- 建立客户与线索、运单的关联关系。
- 支持客户详情页查看历史信息。

## 日志与设置

- 设计登录日志与操作日志结构。
- 明确哪些关键写操作必须记录日志。
- 建立站点设置、上传设置、基础字典等系统设置页面。

## 数据建模

- 明确一期核心对象：
  - `admin_user`
  - `admin_role`
  - `permission`
  - `content_page`
  - `content_block`
  - `news_article`
  - `media_asset`
  - `lead`
  - `lead_follow_record`
  - `customer`
  - `shipment`
  - `shipment_tracking_event`
  - `operation_log`
  - `login_log`
  - `system_setting`
- 避免用万能表或万能 JSON 先凑合。

## 验证与质量

- 建立每个模块最小可用验收标准。
- 确保关键路由、权限、表单、日志都可验证。
- 明确哪些页面需要桌面端优先，哪些页面也要兼顾移动端。
