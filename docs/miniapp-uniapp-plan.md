# uni-app 小程序开发计划

更新时间：2026-05-09

## 目标

基于当前已完成的会员端和支付端后端接口，新增一个 `uni-app` 小程序前端骨架，用于：

- 提前验证会员 API 可用性
- 提前验证小程序登录与支付接入形态
- 为后续正式小程序产品开发提供可运行骨架

## 当前已落地范围

小程序目录：

- `miniapp/`

当前骨架已覆盖：

- 首页
- 手机号注册
- 手机号登录
- 微信登录入口
- 会员资料查看与编辑
- 运单列表
- 运单详情
- 支付记录列表
- 从运单详情发起支付准备
- 支付结果页骨架

## 已适配的后端功能点

- `POST /api/member/auth/register`
- `POST /api/member/auth/login`
- `POST /api/member/auth/wechat-login`
- `GET /api/member/profile`
- `PUT /api/member/profile`
- `PUT /api/member/profile/wechat`
- `GET /api/member/waybills`
- `GET /api/member/waybills/{id}`
- `GET /api/member/payments`
- `PUT /api/member/payments/prepare`

## 当前阶段结论

- 小程序前端骨架已落地
- 类型检查已通过
- 微信小程序构建已打通
- 当前定位仍然是“联调骨架”，不是正式上线版

## 已验证事项

- `npm install` 可完成依赖安装
- `npm run type-check` 可通过
- `npm run build:mp-weixin` 可通过
- 微信小程序导入目录已明确为 `dist/build/mp-weixin`
- 未登录访问受限页面时，已支持登录后回跳原页面
- 支付结果页已支持有限次主动轮询与超时收口
- 支付记录页已支持回看单笔支付结果
- 资料页已补充微信绑定状态摘要和手动刷新
- 首页、运单列表、支付列表已统一状态文案展示
- 小程序内已新增“联调清单”页面，便于在开发者工具里按步骤走链路
- 小程序内已新增“环境诊断”页面，可查看后端地址、登录态和最近接口结果
- 联调清单页已支持本地勾选、进度统计和重置

## 下一阶段计划

### 第一阶段：继续补齐联调闭环

- 在微信开发者工具内完成页面导入和基础预览
- 验证登录态持久化、页面守卫和回跳行为
- 验证运单详情发起支付准备链路
- 校对支付结果页文案、状态分支和超时场景
- 产出开发者工具联调记录

### 第二阶段：补产品化流程

- 细化微信登录后的首次绑手机号流程
- 完善支付成功、失败、取消、处理中反馈页
- 完善支付结果主动轮询退出策略
- 完善错误提示、空状态、超时状态

### 第三阶段：准备真实小程序联调

- 替换真实微信小程序 `appid`
- 配置合法服务器域名
- 在真机环境验证登录和支付
- 补充联调截图、联调记录和操作说明

## 当前明确未完成

- 没有做正式视觉设计还原
- 没有做完整手机号授权产品流程
- 没有做消息通知、订单中心等扩展能力
- 尚未完成微信开发者工具预览留档
- 尚未完成真机支付联调
