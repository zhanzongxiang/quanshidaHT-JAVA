# qsd Admin 项目状态

更新时间：2026-05-09

## 当前阶段判断

项目已完成阶段 0-3，当前处于：

- 阶段 4：微信登录与微信支付扩展，已完成大部分后端与后台管理端落地。
- 阶段 5：测试补齐、联调、上线准备，尚未完成。

当前状态不是“还在搭框架”，而是“主干业务已跑通，支付与微信能力已出第一版，正在收尾和上线化”。

## 已完成

### 1. 后台基础能力

- 管理员登录、JWT 鉴权、菜单、权限、路由守卫已具备。
- 后台基础模块已落地：内容管理、新闻管理、运单管理、字典管理。
- 前端管理端路由和页面已成型，包含会员页、支付页。

### 2. 会员系统

- 会员数据模型与 Flyway 迁移已存在。
- 会员注册、登录、资料查询、资料更新接口已完成。
- 会员 JWT 与管理员 JWT 已隔离。
- 后台会员管理接口与页面已完成。
- 会员与运单的人工绑定、会员查看本人可见运单已完成。

### 3. 微信身份接入

- 会员表已包含 `wechatOpenid`、`wechatUnionid`、`wechatBindTime` 字段。
- 已提供微信登录接口：`POST /api/member/auth/wechat-login`。
- 已提供会员侧微信绑定接口：`PUT /api/member/profile/wechat`。
- 当前实现支持：
  - 通过微信 `code` 换取会话
  - 用 openid 查找会员
  - 首次微信登录自动创建会员，或按手机号绑定已有会员

### 4. 支付系统 v1

- 支付域数据模型已落地：
  - `pay_order`
  - `pay_transaction`
  - `refund_order`
  - `pay_notify_log`
  - `refund_notify_log`
  - `pay_reconcile_record`
- 后台支付管理接口已完成。
- 后台支付管理页面已完成，覆盖：
  - 支付单列表与详情
  - 商户配置管理
  - 发起退款
  - 回调重放
  - 对账记录与差异查看
- 会员侧支付接口已完成：
  - `PUT /api/member/payments/prepare`
  - `GET /api/member/payments`

### 5. 微信支付一期能力

- 小程序支付下单已具备后端实现。
- 微信支付回调、退款回调接口已具备。
- 支持商户配置切换，且支付单会保留商户快照。
- 支持平台证书手动刷新与定时刷新。
- 支持下载交易账单并生成本地对账记录。
- 提供 mock 网关与真实微信网关两套实现。

### 6. 基础验证

- 仓库内已有会员和支付相关控制器测试。
- 文档记录显示：
  - 后端 `mvn -DskipTests package` 已通过
  - 后端 `mvn test` 已通过
  - 前端构建已通过

### 7. 小程序前端骨架

- 仓库已新增 `miniapp/` `uni-app` 小程序骨架。
- 当前已覆盖：
  - 手机号注册与登录
  - 微信登录入口
  - 会员资料查看与编辑
  - 运单列表与详情
  - 支付记录列表

### 2. 真实微信支付上线能力

代码已支持真实微信支付，但默认配置仍是 mock：

- 默认 `WECHAT_PAY_MOCK_ENABLED=true`
- 要切换真实链路，还需要补齐：
  - `appId`
  - `appSecret`
  - `mchId`
  - `notifyUrl`
  - `apiV3Key`
  - `privateKeyPath`
  - `merchantSerialNo`
  - `platformCertificatePath`

也就是说，真实支付“代码通路已具备，生产可用性和联调未收口”。

### 3. 测试覆盖
